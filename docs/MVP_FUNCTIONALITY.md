# MVP функциональность

## Что реализовано

### Публичная лента

Главная страница показывает ленту из MongoDB:

- предложения мероприятий;
- админские посты.

Неавторизованные пользователи видят ленту и могут открыть карточку мероприятия.

### Регистрация и вход

Пользователь может зарегистрироваться, войти и выйти. Backend использует bcrypt для паролей и JWT access token для авторизации.

### Предложения мероприятий

Авторизованный пользователь может предложить мероприятие:

- `title`;
- `external_url`;
- `description`;
- `image_url`.

Изображение можно добавить ссылкой или загрузить из файла в форме создания мероприятия.
Файл сохраняется как `data:image/...;base64,...`, чтобы не добавлять отдельное файловое
хранилище на маленьком VPS.

### Голосование

Авторизованный пользователь может:

- проголосовать за мероприятие;
- снять голос.

Один пользователь может иметь только один голос на одно мероприятие. Это защищено unique index в MongoDB.

### Комментарии

Авторизованный пользователь может комментировать мероприятие.

Неавторизованный пользователь может читать видимые комментарии.

### Админские посты

`admin` и `superadmin` могут создавать посты в ленте.
Только `superadmin` может удалять админские посты.

### Модерация

`admin` и `superadmin` могут:

- менять статус мероприятия;
- скрывать мероприятие;
- скрывать комментарий через API.
- видеть историю действий.

Управление администраторами описано в [AUTH_RBAC.md](AUTH_RBAC.md).

### Управление

`admin` и `superadmin` видят страницу управления пользователями:

- список зарегистрированных пользователей;
- IP регистрации и последнего входа, если известны;
- количество предложенных мероприятий;
- количество голосов;
- мероприятия, за которые пользователь голосовал.

Только `superadmin` может менять роль `user`/`admin`.
Только `superadmin` может очищать историю действий.

### История действий

`admin` и `superadmin` видят историю действий:

- создание мероприятий;
- создание админских постов;
- голоса и снятие голосов;
- комментарии;
- смену статуса и видимости;
- скрытие комментариев;
- удаление мероприятий и админских постов.

При удалении мероприятия или админского поста сохраняется snapshot удаляемой записи.

## Статусы мероприятий

- `proposed` - предложено пользователем;
- `voting` - идет голосование;
- `discussion` - идет обсуждение;
- `accepted` - принято;
- `rejected` - отклонено;
- `completed` - завершено.

Голосование разрешено для:

- `proposed`;
- `voting`;
- `discussion`.

## Где хранятся данные

Postgres:

- пользователи;
- роли;
- audit log.

MongoDB:

- `events`;
- `feed_posts`;
- `votes`;
- `comments`.

MongoDB доступна только внутри Docker-сети и не публикует внешний порт.

## Ручной сценарий в браузере

1. Открыть `https://shiz.booka.dj`.
2. Убедиться, что лента видна без входа.
3. Нажать `Регистрация`, создать пользователя.
4. Нажать `Войти`, войти пользователем.
5. Открыть страницу `Предложить` и отправить предложение мероприятия.
6. Открыть карточку мероприятия в ленте.
7. Проголосовать.
8. Оставить комментарий.
9. Войти под `admin` или `superadmin`.
10. Создать админский пост.
11. Открыть мероприятие и поменять статус.

## Что не реализовано в MVP

- upload изображений;
- отдельные страницы мероприятий;
- сложная модерация комментариев во frontend;
- напоминания о встречах;
- миграции MongoDB;
- refresh token.
